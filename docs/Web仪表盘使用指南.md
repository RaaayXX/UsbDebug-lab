# USBDebug Lab Web 仪表盘 — 用户手册

---

## 1. 概述

USBDebug Lab 是面向 **固件体验测试与 Bug 定位** 的本地 Web 平台：产品经理/测试人员可描述现场异常，平台结合被测设备串口日志、（可选）SmartUSB Hub 电气数据与规则/AI，输出可归档的诊断报告。

### 1.1 主要功能

平台分为 **基础能力**与 **SmartUSB Hub 扩展**（可选）。二者独立：无 Hub 时仍可完成串口监测、规则/AI 分析与日志归档；接入 Hub 后页面自动追加电气监测与控制相关区域。

#### 基础功能（仅串口）

| 功能模块 | 说明 |
|----------|------|
| 串口日志 | 实时显示被测设备串口输出，支持自动滚动与本地保存 |
| 连接配置 | 选择设备日志 COM 口、设备类型与波特率；配置保存在本机 |
| 状态指示 | 顶部显示服务、串口、日志录制状态 |
| 智能分析 | 按设备类型匹配规则关键字；可选接入大模型补充说明 |
| 数据归档 | 停止日志录制时写入 `logs/`；串口区支持带时间戳的手动导出 |
| 测试项目 | 产品名称/介绍本地保存，供 AI 理解被测产品 |

#### SmartUSB Hub 扩展（可选）

需连接 SmartUSB Hub 并在 **应用并连接** 后识别到指令控制口，页面才会显示并启用以下功能：

| 功能模块 | 说明 |
|----------|------|
| 实时监测 | 显示被测 Hub 通道的 VBUS 电压（mV）与电流（mA），并绘制趋势曲线 |
| Hub 控制 | 通道上电/断电、USB 数据通断、硬重启、自检及单次电压/电流采样 |
| 自动化场景 | 按产品类型执行 USB 插拔模拟，并配合曲线与日志录制、分析 |
| 连接配置 | 额外选择 Hub 指令口与被测通道编号 |
| 状态指示 | 额外显示 Hub 指令口连接与曲线采集状态 |
| 智能分析 | 在日志规则基础上，增加电流特征检测与日志—电气交叉分析 |
| 数据归档 | 曲线采集结束时写入 `logs/`；电压/电流区支持 CSV 手动导出 |

### 1.2 设备兼容性

#### 被测设备（串口日志）

平台通过 **USB 虚拟串口（CDC / UART 转 USB）** 采集日志，Web 界面 **设备类型** 下拉框用于选择规则分析所匹配的日志语法；选 **自动识别** 时会根据 COM 口描述与日志内容推断。

**Seeed Studio XIAO 系列（主要兼容目标）**

| 设备类型（界面选项） | 代表型号 | 芯片 | 规则分析覆盖 |
|----------------------|----------|------|--------------|
| XIAO ESP32 | ESP32-S3 / S3 Sense、C3、C5、C6 | Espressif ESP32 | ESP-IDF Panic、看门狗、Brownout、WiFi 等 |
| XIAO SAMD21 | XIAO SAMD21 | Microchip SAMD21 | Arduino 启动、断言、HardFault 等 |
| XIAO RP2040 / RP2350 | XIAO RP2040、RP2350 | Raspberry Pi RP2040 / RP2350 | Pico SDK Panic、MicroPython、看门狗等 |
| XIAO nRF52840 / nRF54L15 | nRF52840 Sense、nRF54L15 Sense | Nordic nRF52 / nRF54 | SDK 错误码、复位原因、BLE 日志等 |
| XIAO RA4M1 | XIAO RA4M1 | Renesas RA4M1 | FSP 错误、Renesas 启动日志等 |
| XIAO MG24 | XIAO MG24 Sense | Silicon Labs EFR32MG24 | SL_STATUS、Matter / Zigbee / Thread 等 |
| 通用 MCU | 其他开发板 | 任意 | 跨平台 HardFault、断言、看门狗等通用规则 |

> XIAO 型号完整列表见 [Seeed XIAO 系列介绍](https://wiki.seeedstudio.com/SeeedStudio_XIAO_Series_Introduction/)。

**其他芯片（如 STM32）**

- **串口监测、日志归档、AI 分析、Hub 电气测试**：可用，手动选择 **通用 MCU** 或 **自动识别** 即可。  
- **规则引擎**：未内置 STM32 HAL / CMSIS 专用关键字，复杂问题建议启用 AI 或自行对照 `serial.log`。  
- **命令行烧录**（`esp32_hub_test.py flash`）：仅支持 ESP32 系列（esptool），STM32 请使用 STM32CubeProgrammer / OpenOCD 等工具。

#### SmartUSB Hub（可选硬件）

| 项目 | 说明 |
|------|------|
| 支持型号 | SmartUSB Hub（指令口 VID `1A86` / PID `FE0C`） |
| 作用 | 与被测芯片无关；控制被测通道 VBUS / D+/D- 并采样电压电流 |
| 适用被测设备 | 任意经 USB 供电或枚举的 XIAO / 开发板 |

### 1.3 路线图（规划）

- **当前**：规则引擎 + 可选 AI；内置 Hub 测试场景；用户填写现象与产品背景  
- **规划**：根据自然语言测试需求，由 AI 生成 Hub 控制脚本并一键执行，再自动汇总日志/曲线生成综合报告  

### 1.4 典型应用

- 固件烧录或升级后是否发生非预期重启  
- 启动阶段日志不完整时的抓取与留存  
- 不同固件版本下的工作电流对比  
- USB 供电稳定性与插拔瞬间电气特征分析

## 2. 环境与硬件

### 2.1 硬件清单

| 设备 | 说明 |
|------|------|
| 主机 | Windows 10/11（本手册命令示例为 PowerShell） |
| 被测开发板 | 推荐 Seeed XIAO 系列；亦支持其他 USB 串口日志设备 |
| SmartUSB Hub | 用于通道电源/数据控制及电压电流采样（可选；无 Hub 时仅可使用串口相关功能） |

### 2.2 Hub 接线说明

SmartUSB Hub 需使用两根 USB 线连接电脑：

| 接口 | 是否需在页面选择 COM | 作用 |
|------|----------------------|------|
| USB 上行口 | 否 | 主机与下游设备的 USB 数据通路；设备枚举与烧录经此接口 |
| 指令控制口 | 是（「Hub 指令口」） | 下发通断电、数据开关指令并读取电压、电流 |

```
                    ┌─────────────────┐
   USB 上行口 ──────│  SmartUSB Hub   │──── 被测通道 ─── 被测开发板（如 XIAO）
   指令控制口 ──────│                 │
                    └─────────────────┘
                            │
                           PC
```

### 2.3 软件安装

进入项目目录后执行（首次安装）：

```powershell
cd <项目目录>
python -m venv venv
.\venv\Scripts\python.exe -m pip install -r requirements.txt
```

PowerShell 激活虚拟环境请使用 `.\venv\Scripts\Activate.ps1`。若脚本执行策略受限，可执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

亦可不激活虚拟环境，直接使用 `.\venv\Scripts\python.exe` 运行服务。

## 3. 启动与访问

### 3.1 启动服务

```powershell
.\venv\Scripts\python.exe dashboard_server.py
```

终端出现 `仪表盘: http://127.0.0.1:8765` 即表示服务已就绪。

### 3.2 打开页面

在浏览器地址栏访问：**http://127.0.0.1:8765**

**重要：** 须通过上述地址访问由本机服务提供的页面；请勿直接双击打开 `web/index.html`，否则无法建立数据连接。

### 3.3 连接状态

页面右上角状态条含义如下：

| 状态项 | 含义 |
|--------|------|
| 服务 | 浏览器与本地调试服务（WebSocket）是否连通 |
| Hub 指令口 | SmartUSB Hub 是否已通过指令控制口连接 |
| 串口 | 应用并连接后设备日志串口是否已打开（可实时显示） |
| 曲线采集 | 测电压/电流或自动化场景时是否在采集 V/I 曲线 |
| 日志录制 | 串口区手动开始录制后是否在写入日志缓冲 |

## 4. 测试项目与连接

### 4.1 测试产品

| 配置项 | 说明 |
|--------|------|
| 产品名称 / 产品介绍 | 选填，写入本机 `user_settings.json`，分析时传给 AI |

未检测到 Hub 时，页面不显示 Hub 相关区域；插入 Hub 并刷新端口或 **应用并连接** 后，电气与场景功能会自动出现。

### 4.2 连接配置

在 **连接配置** 区域完成端口设置后，点击 **应用并连接**（若识别到 Hub 会尝试连接；串口始终按所选日志口打开）。

| 配置项 | 说明 |
|--------|------|
| Hub 指令口 (COM) | Hub 指令控制口对应的串口；留空时应用配置将尝试自动识别 |
| 设备日志口 (COM) | 被测设备串口（列表中带 ★ 的项通常为常见开发板 / XIAO） |
| 设备类型 | 规则分析匹配的芯片系列；**自动识别** 将根据端口与日志推断 |
| Hub 被测口 | 被测设备所接 Hub 通道编号，默认为 1 |
| 波特率 | 与固件一致，常用 115200 |
| 刷新端口 | 重新扫描本机可用 COM 口 |

配置写入本机 `user_settings.json`。分析用 API Key 在右侧 **分析配置** 区域填写，同样仅保存在本机。

## 5. 界面说明

### 5.1 布局概览

```
┌──────────────────────────────────────────────────────────────┐
│  Power Lab    [使用指南]  状态：服务 / Hub / 串口 / 曲线 / 日志  │
├──────────────────────────────────────────────────────────────┤
│  连接配置：设备日志口 · 设备类型 · 波特率 · Hub 指令口 · 被测口   │
├────────────────────────────────────┬─────────────────────────┤
│  电压 · 电流（仪表 + 曲线）          │  快捷指令               │
│  [保存电压/电流]                    │  测试场景 · 分析配置     │
├────────────────────────────────────┴─────────────────────────┤
│  串口输出                          [保存日志] [清空]          │
├──────────────────────────────────────────────────────────────┤
│  分析结果                          [待分析 / 正常 / 警告]    │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 电压与电流

| 元素 | 说明 |
|------|------|
| VBUS 仪表 | 当前通道电压，单位 mV（约 5000 mV 对应 5 V） |
| 电流仪表 | 当前通道电流，单位 mA |
| 趋势曲线 | 蓝色为电压，绿色为电流；横轴为相对录制起点的毫秒时间 |
| 保存数据 | 标题旁按钮：分别生成 `*_voltage.csv`、`*_current.csv`（有数据则导出；曲线与单次读数合并于对应文件，含时间戳） |
| 告警提示 | 电流持续低于约 30 mA 时提示，可能表示掉电或重启（阈值可在 `config.yaml` 中 `reboot_current_threshold_ma` 调整） |

曲线在 **测电压**、**测电流** 或运行 **测试场景** 时由 Hub 自动周期性采样；未连接 Hub 时仪表与曲线不可用。

### 5.3 快捷指令

用于对当前 Hub 被测通道进行单步操作，与 **产品类型** 无关。标题旁 **?** 可查看各指令说明。

| 指令 | 功能 |
|------|------|
| 上电 / 断电 | 控制通道 VBUS 电源 |
| 连接数据 / 断开数据 | 控制 USB D+/D- 数据通路 |
| 硬重启 | 通道断电后重新上电 |
| Hub 自检 | 执行断电—上电序列，用于验证 Hub 与控制链路 |
| 测电压 / 测电流 | 单次读取并自动开始曲线采集（若尚未采集） |

### 5.4 测试场景

先选择 **产品类型**，再执行对应场景。自动化插拔序列因产品类型而异：

| 产品类型 | 场景中的插拔序列 |
|----------|------------------|
| 带电池产品 | 断电并断开数据 → 上电并连接数据（模拟 USB 线缆整体拔出与接入） |
| USB 供电开发板 | 断电 → 上电 |

各场景名称旁 **?** 提供该场景的步骤与判定说明。

### 5.5 串口输出

- **应用并连接** 后实时显示设备串口打印（无需先点录制）。  
- **日志录制**：开始/停止将此后输出写入缓冲，停止时保存到 `logs/`；运行测试场景时会自动开始日志录制。  
- **自动滚动**：开启后视图跟随最新输出。  
- **保存日志**：导出当前显示内容；文件头为连接元数据，**每行日志**前缀 `[接收时间]`。  
- **清空**：仅清除页面显示，不影响已导出或已停止录制写入磁盘的文件。  

### 5.6 分析配置与结果

在 **分析配置**（侧栏）中可填写 OpenAI 兼容 API Key 与接口地址；留空 Key 时仅执行规则分析。

在 **分析结果** 区域：

| 元素 | 说明 |
|------|------|
| 观察到的现象 | 选填。描述现场异常（如黑屏、反复重启、电流偏高）；将与当前串口日志、电压/电流曲线一并送入规则与 AI |
| 智能分析 | 综合用户描述、日志关键字、电气特征生成结论；现象描述会写入诊断报告 |

点击 **智能分析** 后展示：

- **诊断摘要**：整体结论  
- **异常发现**：每项包含异常描述、发现来源、可能原因、建议排查及依据片段  
- **AI 补充说明**（已配置且启用时）：自然语言补充  

| 结果标签 | 含义 |
|----------|------|
| 正常 | 未命中需重点关注的规则项 |
| N 项警告 | 存在重启迹象、欠压或电流异常等 |
| N 项严重 | 存在 Panic、看门狗或断言等严重错误 |

点击 **保存全部（曲线+测试记录+日志）** 将生成完整诊断包，目录位于 `logs/YYYYMMDD_HHMMSS_report/`，包含：

| 文件 | 内容 |
|------|------|
| `*_report.md` | 可读诊断报告（摘要、各项异常及排查建议） |
| `analysis.json` | 结构化分析结果 |
| `*_power.csv` | 电压/电流曲线 |
| `*_measure.csv` | 单次采样记录（如有） |
| `*_serial.log` | 串口日志 |

若保存时尚未执行分析，系统将先根据当前数据自动完成规则分析再写入报告。

## 6. 操作指南

### 6.1 USB 插拔与重启诊断

1. 配置 Hub 指令口、设备日志口、**设备类型** 及 **产品类型**，点击 **应用并连接**。  
2. 确认顶部 **服务**、**Hub 指令口**、**串口** 为正常状态。  
3. 在 **测试场景** 中执行 **USB 插拔 + 重启诊断**（将自动采集曲线与日志）。  
4. 观察进度条、曲线、串口输出及底部分析结论。  

### 6.2 烧录后重启观测

1. 完成连接配置并 **应用并连接**。  
2. 在串口区 **开始录制** 日志；需要曲线时可先 **测电流** 或运行场景以自动采集。  
3. 在独立终端执行固件烧录（若与 Web 共用同一 COM，须先 **应用并连接** 断开或关闭服务以释放串口）。  
4. 根据曲线跌落、串口二次启动信息及页面提示判断行为。  
5. 停止日志录制后点击 **智能分析** 生成报告。  

### 6.3 Hub 功能验证

通过 **Hub 自检** 或 **硬重启** 观察仪表读数与提示信息，确认指令控制口通信正常。

### 6.4 未接 Hub（仅串口）

未插入 SmartUSB Hub 时，页面自动隐藏 Hub 相关区域。流程：

1. 选好 **设备日志口** 与 **设备类型**，点 **应用并连接**。  
2. 串口区实时显示日志；需要归档时 **开始录制** / **停止**。  
3. 填写 **观察到的现象**，执行 **智能分析**。

### 6.5 固件电流对比

1. **应用并连接** 后点击 **测电流**（或运行含采样的场景），待设备完成启动及典型负载稳定。  
2. 停止曲线采集（结束场景或重启服务前会自动保存），保留 `logs/` 下 `*_vi/power.csv`。  
3. 更换固件后重复上述步骤，对比峰值与稳态电流。

## 7. 数据文件说明

所有归档目录位于项目 `logs/` 下。

### 7.1 停止采集（自动写入）

曲线与日志分目录保存，例如：

- `logs/20260603_143022_vi/power.csv` — 曲线采集结束或 VI 线程停止时写入  
- `logs/20260603_143055_serial/serial.log` — 串口区 **停止** 日志录制时写入  

| 文件 | 内容 |
|------|------|
| `power.csv` | 列：`wall_time`、`t_ms`、`v_mV`、`i_mA`；含保存时间与采集开始时间注释 |
| `serial.log` | 含元数据头；正文每行 `[wall_time] 日志内容` |

### 7.2 分区手动保存

| 区域 | 操作 | 输出 |
|------|------|------|
| 电压·电流 | 保存数据 | `logs/…_export/` 下的 `*_voltage.csv`、`*_current.csv`（各一份，有数据则生成） |
| 串口输出 | 保存日志 | `logs/…_export/*_serial.log`（每行 `[时间] 日志内容`） |
| 分析结果 | 保存全部（曲线+测试记录+日志） | `logs/…_report/` 诊断报告包（见 5.6 节） |

CSV 文件可用 Excel 等工具打开作图；`report.md` 可直接用于测试记录与问题归档。

## 8. 常见问题

### Q1：页面显示服务未连接

- 确认 `dashboard_server.py` 正在运行且无报错。  
- 访问地址为 `http://127.0.0.1:8765`（默认端口，可在 `config.yaml` 的 `dashboard.port` 中修改）。  
- 检查本机防火墙是否拦截该端口。  

### Q2：串口无输出

- 确认 **设备日志口** 与设备管理器中 COM 号一致。  
- 确认串口未被其他程序占用（如 `idf.py monitor`、串口助手）。  
- 确认固件经 USB CDC 输出日志；若日志仅走 UART 引脚，需使用对应转串口设备并选择正确 COM。  

### Q3：电压/电流显示为「—」

- 确认 Hub 上行口与指令控制口均已连接。  
- 确认 **Hub 指令口** 选择正确，或留空后由服务自动扫描。  
- 确认被测设备接在配置的 **Hub 被测口** 上。  

### Q4：Hub 指令无响应

- 查看状态条 **Hub 指令口** 及页面提示。  
- 检查 Hub 供电与线缆连接后，重启 `dashboard_server.py` 并重新 **应用并连接**。  

### Q5：分析结果为「正常」但现象异常

- 规则分析依赖日志关键字与电流特征，未输出典型错误时可能漏报。  
- 建议配置 API Key 启用 AI 补充，或结合 `serial.log` 人工排查。  
- 分析前宜在串口区 **开始录制** 或运行场景，积累足够日志后再执行 **智能分析**。  

### Q6：烧录与 Web 串口冲突

- 同一 COM 口不宜同时用于烧录与 Web 采集。  
- 建议：烧录前关闭 Web 串口占用 → 烧录完成后再 **应用并连接** 并 **开始录制** 观察启动。  

### Q7：修改配置后未生效

- 修改 `config.yaml` 或端口列表后，需终止并重新启动 `dashboard_server.py`。  
- 网页内 COM 与产品类型以 **应用并连接** 后写入的 `user_settings.json` 为准。

## 9. 安全与注意事项

- 服务默认监听 `127.0.0.1`，仅本机可访问。若需局域网访问，须明确安全风险后再修改 `dashboard.host`。  
- **断电**、**硬重启** 会中断正在进行的烧录或文件写入，请勿在 Flash 操作过程中执行。  
- 大电流负载请按 Hub 产品说明连接辅助供电。  
- API Key 为敏感信息，请勿提交至版本库；仅保存在本机配置文件中。
