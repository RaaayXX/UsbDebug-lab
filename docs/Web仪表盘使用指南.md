# USBDebug Lab Web 仪表盘 — 用户手册

---

## 1. 概述

USBDebug Lab 是本地 Web 调试平台，用于采集被测设备串口日志，可选接入 SmartUSB Hub 采集电气数据，并通过规则引擎与 AI 生成诊断报告。

### 1.1 功能

| 模块 | 无 Hub | 有 Hub |
|------|--------|--------|
| 串口日志 | ✓ | ✓ |
| 设置（端口、设备类型、产品、API） | ✓ | ✓ |
| 智能分析 | ✓ | ✓ |
| 电压 · 电流 | — | ✓ |
| 快捷指令 | — | ✓ |
| 测试场景 | — | ✓ |

### 1.2 设备类型（规则分析）

| 界面选项 | 芯片平台 |
|----------|----------|
| 自动识别 | 按 COM 描述与日志推断 |
| ESP32（S3 / C3 / C5 / C6） | Espressif ESP32 |
| SAMD21 | Microchip SAMD21 |
| RP2040 / RP2350 | Raspberry Pi RP2040 / RP2350 |
| nRF52840 / nRF54L15 | Nordic nRF52 / nRF54 |
| RA4M1 | Renesas RA4M1 |
| EFR32 MG24 | Silicon Labs EFR32MG24 |
| 通用 MCU（其他芯片） | 跨平台通用规则 |

未列出的芯片选用 **通用 MCU（其他芯片）** 或 **自动识别**。命令行烧录（`esp32_hub_test.py flash`）仅支持 Espressif ESP32。

### 1.3 SmartUSB Hub

| 项目 | 说明 |
|------|------|
| 型号 | VID `1A86` / PID `FE0C` |
| 功能 | 通道 VBUS / D+/D- 控制，电压电流采样 |

## 2. 环境与安装

| 项目 | 要求 |
|------|------|
| 主机 | Windows 10/11 |
| 被测设备 | USB 串口日志输出 |
| SmartUSB Hub | 可选 |

Hub 接线：上行口接 PC；指令控制口在 **Hub 指令口** 中选择对应 COM。

```powershell
cd <项目目录>
python -m venv venv
.\venv\Scripts\python.exe -m pip install -r requirements.txt
```

---

## 3. 启动与访问

```powershell
.\venv\Scripts\python.exe dashboard_server.py
```

浏览器访问 **http://127.0.0.1:8765**。须通过上述地址访问，不可直接打开 `web/index.html`。

### 状态条

| 状态项 | 含义 |
|--------|------|
| 服务 | WebSocket 连接 |
| Hub 指令口 | Hub 指令控制口连接 |
| 串口 | 设备日志串口连接 |
| 曲线采集 | V/I 曲线采集中 |
| 场景日志 | 测试场景日志录制中 |

## 4. 设置

点击 **应用并连接** 保存配置并连接。配置写入 `user_settings.json`。

| 配置项 | 说明 |
|--------|------|
| 设备日志口 | 被测设备 COM 口 |
| 设备类型 | 规则分析芯片系列 |
| 波特率 | 串口波特率 |
| Hub 指令口 | Hub 指令控制口；留空自动识别 |
| 被测通道 | Hub 通道 1–4 |
| 刷新端口 | 重新扫描 COM 口 |
| 应用并连接 | 保存并连接 |
| 断开 Hub | 断开 Hub 连接 |
| 产品名称 | 产品标识 |
| 产品背景 | 上传 `.md` 文件 |
| API Key | OpenAI 兼容 API 密钥 |
| API 端点 | API 地址 |
| 模型 | 模型名称 |
| 启用 AI | 分析时调用 AI |

未检测到 Hub 时，页面隐藏 Hub 相关区域。

## 5. 界面

### 5.1 布局

```
┌──────────────────────────────────────────────────────────────┐
│  USBDebug Lab  [使用指南]  服务 / Hub 指令口 / 串口 / 曲线 / 场景日志 │
├──────────────────────────────────────────────────────────────┤
│  设置                                                          │
├────────────────────────────────────┬─────────────────────────┤
│  电压 · 电流  [保存数据]             │  快捷指令               │
│                                    │  测试场景               │
├────────────────────────────────────┴─────────────────────────┤
│  串口输出  [自动滚动] [保存日志] [清空]                          │
├──────────────────────────────────────────────────────────────┤
│  分析结果  [待分析]  [保存报告与日志]                            │
└──────────────────────────────────────────────────────────────┘
```

有 Hub 时，保存按钮显示为 **保存全部（曲线+测试记录+日志）**。

### 5.2 电压 · 电流

| 元素 | 说明 |
|------|------|
| VBUS | 通道电压（mV） |
| 电流 mA | 通道电流（mA） |
| 趋势曲线 | 电压（蓝）、电流（绿） |
| 保存数据 | 导出 `*_voltage.csv`、`*_current.csv` |

由 **测电压**、**测电流** 或 **测试场景** 触发采集。

### 5.3 快捷指令

| 指令 | 说明 |
|------|------|
| 上电 | 打开 VBUS 5V |
| 断电 | 关闭 VBUS 5V |
| 连接数据 | 接通 D+ / D- |
| 断开数据 | 断开 D+ / D- |
| 硬重启 | 断电 → 上电 |
| Hub 自检 | 断电 → 上电 |
| 测电压 | 开始/停止 VBUS 曲线 |
| 测电流 | 开始/停止电流曲线 |

### 5.4 测试场景

| 配置项 | 说明 |
|--------|------|
| 产品类型 | 带电池 / 不带电池 |
| 测试场景 | 见下表 |
| 开始测试 | 执行所选场景 |

| 场景 | 产品类型 | 参数 |
|------|----------|------|
| USB 插拔 + 重启诊断 | 带电池、不带电池 | 重复插拔次数、插拔间隔 (秒) |
| 仅电池供电 + 串口监测 | 带电池 | 重复轮数、仅电池观测时长 (秒) |

| 产品类型 | 插拔序列 |
|----------|----------|
| 带电池 | 断电+断开数据 → 上电+连接数据 |
| 不带电池 | 断电 → 上电 |

### 5.5 串口输出

| 操作 | 说明 |
|------|------|
| 自动滚动 | 跟随最新输出 |
| 保存日志 | 导出 `*_serial.log` |
| 清空 | 清除页面显示 |

**应用并连接** 后实时显示日志。运行测试场景时自动录制场景日志。

### 5.6 分析结果

| 元素 | 说明 |
|------|------|
| 观察到的现象 | 现场异常描述 |
| 智能分析 | 执行规则与 AI 分析 |
| 待分析 / 正常 / N 项警告 / N 项严重 | 分析状态 |
| 保存报告与日志 | 导出诊断报告（无 Hub） |
| 保存全部（曲线+测试记录+日志） | 导出完整数据包（有 Hub） |
| 完整 AI 回复 | AI 原始输出 |

分析输出：诊断摘要、异常发现（描述、来源、原因、建议、依据）、AI 补充说明。

报告目录 `logs/YYYYMMDD_HHMMSS_report/`：

| 文件 | 内容 |
|------|------|
| `*_report.md` | 诊断报告 |
| `analysis.json` | 结构化结果 |
| `*_power.csv` | 电压/电流曲线 |
| `*_measure.csv` | 采样记录 |
| `*_serial.log` | 串口日志 |

## 6. 操作流程

### 6.1 USB 插拔 + 重启诊断

1. **设置** → 配置端口、设备类型、产品类型 → **应用并连接**
2. 选择场景 **USB 插拔 + 重启诊断** → **开始测试**
3. 查看曲线、串口输出、分析结果

### 6.2 仅电池供电 + 串口监测

1. 产品类型选 **带电池**
2. 选择场景 **仅电池供电 + 串口监测** → **开始测试**

### 6.3 烧录后观测

1. **应用并连接**
2. 烧录前释放 COM 口（断开连接或关闭服务）
3. 烧录完成后重新 **应用并连接**
4. **测电流** 或运行场景采集曲线
5. **智能分析** → **保存报告与日志**

### 6.4 仅串口（无 Hub）

1. 配置 **设备日志口**、**设备类型** → **应用并连接**
2. 填写 **观察到的现象** → **智能分析** → **保存报告与日志**

## 7. 数据文件

目录：`logs/`

| 触发方式 | 输出 |
|----------|------|
| 曲线采集结束 | `…_vi/power.csv` |
| 保存数据 | `…_export/*_voltage.csv`、`*_current.csv` |
| 保存日志 | `…_export/*_serial.log` |
| 保存报告 | `…_report/` 诊断包 |

`power.csv` 列：`wall_time`、`t_ms`、`v_mV`、`i_mA`。

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| 服务未连接 | 确认 `dashboard_server.py` 运行；访问 `http://127.0.0.1:8765` |
| 串口无输出 | 核对 COM 口与波特率；确认端口未被占用 |
| 电压/电流为「—」 | 确认 Hub 接线；核对 **Hub 指令口** 与 **被测通道** |
| Hub 无响应 | 检查 **Hub 指令口** 状态；重启服务后 **应用并连接** |
| 分析结果为正常 | 配置 API Key 启用 AI；运行场景积累日志后重新分析 |
| 烧录与串口冲突 | 烧录前释放 COM 口 |

---

## 9. 注意事项

- 服务默认监听 `127.0.0.1`
- **断电**、**硬重启** 勿在烧录过程中执行
- API Key 仅存于本机，勿提交版本库
